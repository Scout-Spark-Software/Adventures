# URL Slugs for Hikes, Camping Sites, and Backpacking

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add human-readable, unique URL slugs to all three content types so URLs look like `/hikes/sunset-trail` instead of `/hikes/3f2b...uuid`.

**Architecture:** Add a `slug` text column (unique, not null) to each of the three tables. Generate slugs from the entity name on create, regenerate on rename. The API resolves `[id]` params as UUID first (backward compat), falling back to slug lookup. UI links switch to using `slug`. A migration script backfills slugs for all existing rows. Profanity is checked before the slug is accepted.

**Tech Stack:** Drizzle ORM (Neon Postgres), SvelteKit API routes, existing `obscenity`-based profanity filter at `src/lib/utils/profanity-filter.ts`, `npx tsx` for migration scripts.

```ts
// src/lib/utils/slugify.ts
import { checkProfanity } from "./profanity-filter";

/**
 * Convert a name to a URL-safe slug.
 * e.g. "Sunset Trail (WA)" → "sunset-trail-wa"
 */
export function toSlug(text: string): string {
  return text
    .toLowerCase()
    .trim()
    .replace(/[^a-z0-9\s-]/g, " ")
    .trim()
    .replace(/[\s-]+/g, "-")
    .replace(/^-+|-+$/g, "");
}

/**
 * Returns true if the slug (or the name it was derived from) contains profanity.
 */
export function hasProfanityInSlug(slug: string): boolean {
  // Replace hyphens with spaces so the profanity matcher reads words correctly
  const readable = slug.replace(/-/g, " ");
  return checkProfanity(readable).hasProfanity;
}
```

### Task 2: Server-side unique slug generator

**Files:**
- Create: `src/lib/server/slug.ts`

**Step 1: Implement**

This function generates a slug from a name, checks for profanity, then queries the DB to ensure uniqueness. If the base slug is taken it appends `-2`, `-3`, etc.

```ts
// src/lib/server/slug.ts
import { eq, and, ne } from "drizzle-orm";
import { db } from "$lib/db";
import { hikes, campingSites, backpacking } from "$lib/db/schemas";
import { toSlug, hasProfanityInSlug } from "$lib/utils/slugify";
import { error } from "@sveltejs/kit";

type SlugTable = "hike" | "camping_site" | "backpacking";

const tableMap = {
  hike: hikes,
  camping_site: campingSites,
  backpacking: backpacking,
} as const;

/**
 * Generate a unique, profanity-free slug for an entity.
 *
 * @param name        The human-readable name (e.g. "Sunset Trail")
 * @param entityType  Which table to check uniqueness against
 * @param excludeId   When renaming an existing entity, pass its own ID so it
 *                    doesn't conflict with itself
 */
export async function generateUniqueSlug(
  name: string,
  entityType: SlugTable,
  excludeId?: string,
): Promise<string> {
  const base = toSlug(name);

  if (!base) {
    throw error(400, "Name produces an empty slug");
  }

  if (hasProfanityInSlug(base)) {
    throw error(400, "Name contains inappropriate language");
  }

  const table = tableMap[entityType];

  // Try the base slug, then base-2, base-3, …
  let candidate = base;
  let counter = 2;

  while (true) {
    const conditions = excludeId
      ? and(eq(table.slug, candidate), ne(table.id, excludeId))
      : eq(table.slug, candidate);

    const existing = await db.query[entityType === "hike" ? "hikes" : entityType === "camping_site" ? "campingSites" : "backpacking"].findFirst({
      where: conditions,
      columns: { id: true },
    });

    if (!existing) return candidate;

    candidate = `${base}-${counter++}`;
  }
}
```

### Task 4: Backfill slugs for existing records

**Files:**
- Create: `scripts/migrate-slugs.ts`

**Step 1: Write the backfill script**

```ts
// scripts/migrate-slugs.ts
import { neon } from "@neondatabase/serverless";
import dotenv from "dotenv";

dotenv.config();

const sql = neon(process.env.DATABASE_URL!);

/**
 * Minimal slug conversion (mirrors src/lib/utils/slugify.ts — no imports
 * available from scripts context without bundling).
 */
function toSlug(text: string): string {
  return text
    .toLowerCase()
    .trim()
    .replace(/[^a-z0-9\s-]/g, " ")
    .trim()
    .replace(/[\s-]+/g, "-")
    .replace(/^-+|-+$/g, "");
}

async function backfillTable(tableName: string) {
  const rows = await sql(`SELECT id, name FROM ${tableName} WHERE slug IS NULL`);
  console.log(`${tableName}: ${rows.length} rows to backfill`);

  const slugCounts: Record<string, number> = {};

  // Pre-load existing slugs to avoid conflicts with already-named rows
  const existingSlugs = await sql(`SELECT slug FROM ${tableName} WHERE slug IS NOT NULL`);
  for (const { slug } of existingSlugs) {
    slugCounts[slug] = (slugCounts[slug] ?? 0) + 1;
  }

  for (const row of rows) {
    const base = toSlug(row.name) || row.id;
    let candidate = base;
    let counter = 2;

    while (slugCounts[candidate]) {
      candidate = `${base}-${counter++}`;
    }

    slugCounts[candidate] = 1;

    await sql(`UPDATE ${tableName} SET slug = $1 WHERE id = $2`, [candidate, row.id]);
    console.log(`  ${row.name} → ${candidate}`);
  }
}

async function main() {
  await backfillTable("hikes");
  await backfillTable("camping_sites");
  await backfillTable("backpacking");
  console.log("\nDone.");
}

main().catch((e) => {
  console.error(e);
  process.exit(1);
});
```

### Task 10: API GET — resolve by slug or UUID

**Files:**
- Modify: `src/routes/api/hikes/[id]/+server.ts`
- Modify: `src/routes/api/camping-sites/[id]/+server.ts`
- Modify: `src/routes/api/backpacking/[id]/+server.ts`

Currently each GET handler queries `WHERE id = params.id` (UUID). We need to also accept a slug so old UUID-based bookmarks keep working and new slug-based URLs work too.

**Step 1: Add a UUID detection helper — inline, no new file needed**

A valid UUID v4 matches: `/^[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}$/i`

**Step 2: Update hikes GET handler**

Find the line that reads (approximately):
```ts
.where(eq(hikes.id, params.id))
```

Replace the entire where clause with a helper that queries by UUID if it looks like one, otherwise by slug:

```ts
const UUID_RE = /^[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}$/i;
const isUuid = UUID_RE.test(params.id);
// ...
.where(isUuid ? eq(hikes.id, params.id) : eq(hikes.slug, params.id))
```

The `or()` approach would also work but the UUID-first check is cleaner and avoids unnecessary OR conditions.

**Step 3: Apply the same change to camping-sites and backpacking GET handlers**

Same `UUID_RE` check, `eq(campingSites.slug, params.id)` / `eq(backpacking.slug, params.id)` as fallback.
